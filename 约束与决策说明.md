# 约束与决策说明

本项目优先交付可运行、可解释、可验证的 Android 端画质分析工具，因此采用原生 Android + Java + XML，不接入深度学习模型和 OpenCV，降低模型体积、训练数据、依赖配置和答辩解释成本。

图片解码依赖 Android 系统能力。JPEG、PNG、WebP 作为核心格式；HEIC/HEIF、AVIF、BMP、GIF 作为扩展尝试格式，其中 HEIC/AVIF 受系统版本影响，GIF 只分析首帧。大图会缩放后分析，目标是避免 OOM 并提升响应速度。

评分采用清晰度、曝光、噪点、对比度、色偏、色彩丰富度、信息熵 7 个维度。综合分用于工程筛查，不等同于审美分。低纹理、夜景、强滤镜、景深虚化等场景可能误判，因此保留反例并说明局限。
