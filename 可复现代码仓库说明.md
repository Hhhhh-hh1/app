# 可复现代码仓库说明

## 目标

本文件说明如何从当前仓库复现图像画质分析工具，覆盖源码、依赖、运行命令、验证材料和不应提交的文件范围。它对应“可复现的代码仓库”要求，重点是让别人能看懂、能构建、能追溯结果。

## 仓库内容

- `app/src/main/java/com/example/imagequality/`：Android 主程序和画质分析核心代码。
- `app/src/main/res/`：界面布局、颜色、图标和样式资源。
- `app/src/test/`：本地单元测试。
- `README.md`：目标、非目标、完成判据、运行步骤和算法说明。
- `约束与决策说明.md`：工程取舍、限制和评分边界。
- `AI协作说明.md`：AI 生成、人工确认和关键路径角色说明。
- `验证证据/`：构建记录、测试样本、对比表和反例分析。
- `.gitignore`：排除构建产物、缓存、日志、压缩包和本地环境文件。

## 复现环境

- 操作系统：Windows 环境。
- Android Studio：已安装并可打开 Gradle 项目。
- JDK：建议使用 Android Studio 自带 JBR，路径示例为 `D:\Program Files\Android\Android Studio\jbr`。
- Android SDK：路径示例为 `D:\Android\Sdk`。
- Gradle Wrapper：使用仓库内 `gradlew.bat`，不要求全局安装 Gradle。
- Android 配置：`compileSdk 35`、`minSdk 26`、`targetSdk 35`。

## 复现步骤

1. 克隆或解压当前仓库。
2. 使用 Android Studio 打开仓库根目录。
3. 等待 Gradle 同步完成。
4. 在 PowerShell 中设置 JDK：

```powershell
$env:JAVA_HOME="D:\Program Files\Android\Android Studio\jbr"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
```

5. 执行构建和测试：

```powershell
.\gradlew.bat testDebugUnitTest assembleDebug
```

6. 构建成功后查看 APK：

```text
app/build/outputs/apk/debug/app-debug.apk
```

7. 连接真机后安装：

```powershell
adb devices
adb install -r app\build\outputs\apk\debug\app-debug.apk
```

8. 打开 App，选择 `验证证据/测试样本/` 中的图片或手机相册图片，记录综合分和子项分。

## 结果追溯

- 构建结果应记录到 `验证证据/构建验证记录.md`。
- 样本评分应记录到 `验证证据/README.md` 或 `验证证据/第三_四点测试报告.md`。
- 若发现评分失准，应保留图片、人工判断、App 结果和原因分析。
- 当前仓库已有反例方向：景深虚化照片可能被全局清晰度算法误判。

## Git 提交注意事项

`.gitignore` 已排除 `.gradle/`、`build/`、`app/build/`、`.idea/`、`local.properties`、APK/AAB、日志、压缩包、临时目录、缓存目录和输出结果目录。提交仓库时建议只保留源码、说明文件和必要验证样本，避免把本地环境、构建产物和大体积中间文件放入 Git。

本文件只说明复现流程，不代表本次已经重新完成真机构建和实测。最终提交前建议重新运行完整构建命令，并补充至少一张 App 评分截图和一份样本对比表。
